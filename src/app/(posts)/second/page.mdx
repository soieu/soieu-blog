# 블로그에 오신 것을 환영합니다!

This article explains what the MDX format is. It shows how markdown, JSX, JavaScript expressions, and import and export statements in ESM can be used inside MDX. See § Getting started for details on how to integrate MDX into your project. See § Using MDX when you’re already all set up to use MDX.

- 동적인 JSX 컴포넌트
- 완전 반응형 디자인
- 간단한 커스터마이징

## MDX란 무엇인가요?

MDX는 Markdown과 React 컴포넌트를 하나의 파일에서 사용할 수 있게 해줍니다. 예를 들어:

<Highlight>"MDX는 Markdown과 JSX의 장점을 결합한 도구입니다!"</Highlight>

### 왜 Tailwind CSS인가요?

Tailwind CSS를 사용하면 유틸리티 클래스를 통해 간단하게 스타일링할 수 있습니다. 예를 들어:

<CodeBlock>
  {`<div className="bg-blue-500 text-white p-4 rounded-lg">
  안녕하세요, Tailwind입니다!
</div>`}
</CodeBlock>

#### Tailwind CSS의 장점

1. 빠르게 스타일링 가능
2. 높은 커스터마이징 가능
3. React 및 MDX와의 완벽한 호환성

---

### 코드 예제

JavaScript로 간단한 함수를 정의하는 방법은 다음과 같습니다:

<CodeBlock>
  {`function greet(name) {
  return \`안녕하세요, \${name}!\`;
}`}
</CodeBlock>

#### 결과

`greet("MDX")`를 호출하면 다음과 같은 결과가 반환됩니다:
